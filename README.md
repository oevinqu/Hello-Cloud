## 📂 저장소 구조 및 규칙

### 1. 기본 구조
각자의 이름 폴더(`Name`)를 최상위로 두고, 내부 관리는 본인의 스타일에 맞춰 **자유롭게** 진행합니다.

* **추천 방식 A (날짜 우선):** `Name/YYYY-MM-DD/Subject/파일명.md`
  * 예: `Jeon-HyoBeen/2026-01-19/AWS/IAM_정리.md`
* **추천 방식 B (주제 우선):** `Name/Subject/YYYY-MM-DD/파일명.md`
  * 예: `MemberA/AWS/2026-01-19/네트워크_이론정리.md`

> 💡 **Tip:** 어떤 방식을 선택하든, 파일명에 **핵심 주제**를 포함해 주세요! (나중에 검색하기 편합니다.)

### 1-1. 폴더 구조 예시
```text
Hello-Cloud/
├── 📂 Jeon-HyoBeen/          # [방법 A] 날짜 폴더 안에 주제별 분류
│   └── 📂 2026-01-19/
│       ├── 📂 AWS/
│       │   └── IAM_정리.md
│       └── 📂 BOJ/
│           ├── BOJ_1000.py
│           └── BOJ_9039.py
├── 📂 MemberA/               # [방법 B] 주제 폴더 안에 날짜별 분류
│   └── 📂 AWS/
│       └── 📂 2026-01-19/
│           └── 네트워크_이론정리.md
└── 📂 MemberB/               # [방법 C] 날짜 폴더 안에 파일 바로 업로드
    └── 📂 2026-01-19/
        ├── 인프런_CSS강의정리.md
        ├── AWS_IAM정리.md
        └── BOJ_1000.py
```

### 2. 기록 가이드
* **이론 공부:** 마크다운 형식(`.md`)을 권장합니다.
* **코딩 테스트:** 본인이 사용하는 언어의 확장자(`.py`,`.java`,`.cpp` 등)로 자유롭게 업로드해주세요.
* **이미지:** 이미지가 필요한경우 마크다운에 직접 첨부하거나, 해당 주차 폴더 내에 업로드해주세요.
* **파일명 규칙:** 파일명에 **공백(Space) 대신 언더바 (`_`) 혹은 하이픈 (`-`)** 을 사용해 주세요!!
* * ✅ 좋은 예: `AWS_SAA_Week1.md`, `BOJ_1000.py`
  * ❌ 피해야 할 예: `AWS 네트워크 정리.md` (공백 포함)

### 3. 업로드 및 브랜치 규칙 (중요!)
* **개인 브랜치 사용:** `main` 브랜치에 직접 Push 하지 마시고, **본인 이름의 브랜치**를 생성해 작업해 주세요! (예: `member/hyobeen`)
    * 매일의 학습 기록은 본인 브랜치에 자유롭게 `push` 하여 잔디를 심으시면 됩니다. 🌱
* **주간 PR (Pull Request):** 매주 일요일 밤, 일주일간의 기록을 `main` 브랜치로 합치는 **PR**을 날려주세요. 스터디장(전효빈)이 확인 후 매주 월요일에 일괄 머지(Merge)합니다.
* **‼️개인 영역 외 작업 금지‼️:** 공유 레포이므로, 반드시 **본인 폴더 내에서만** 작업해 주세요!
    * 실수로라도 다른 팀원의 작업물을 수정/삭제하지 않도록 주의 부탁드립니다.
    * 공용 파일(`README.md`, `.gitignore` 등) 수정이 필요한 경우, 스터디장(전효빈)에게 먼저 말씀해 주세요!
* **실수는 괜찮아요:** 만약 작업 중 꼬이거나 실수로 다른 파일을 건드렸다면 당황하지 말고 저에게 바로 알려주세요. 금방 복구해 드릴게요! 
* **함께 성장해요:** 모르는 게 생기면 언제든 연락 주세요. 같이 고민하고 해결해 보아요! 🔥

### 4. 커밋 메시지 규칙
* 날짜와 함께, 어떤 작업을 했는지 짧게 남겨주세요!
* **Add:** 새로운 학습 기록/코드 추가
* **Edit:** 기존 내용 수정 및 보완
* (예시: `Add: 1월 19일 백준 문제 풀이`, `Edit: Git 이론 강의 노트 수정`)

## 📢 학습 기록 방식 및 개인 채널 공유 안내

우리 스터디는 **기록의 자율성**을 존중합니다! 

1. **기록 도구 자율:** 꼭 깃허브가 아니어도 좋습니다. 개인 블로그, 노션, 티스토리 등 본인이 편한 곳에 기록하셔도 무방합니다.
2. **링크 공유:** 개인 채널에 학습 기록을 남기시는 분들은 **저(전효빈)에게 링크를 보내주세요!** 아래 [스터디원 현황] 표에 연결해 드립니다.
3. **그래도 깃허브를 추천하는 이유:** * 우리 스터디의 공식 저장소에 발자취가 남습니다.
   * 서로의 공부 내용을 가장 쉽게 구경하고 피드백을 주고받을 수 있습니다.
   * (권장) 외부 채널을 쓰시더라도, 주간 회고록 정도만 깃허브에 남겨주시면 잔디도 심고 소통하기 더 좋습니다! 😊
