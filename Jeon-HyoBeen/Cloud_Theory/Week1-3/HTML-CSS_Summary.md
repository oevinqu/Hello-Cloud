## 인프라 및 DB 기초
- 프록시 서버 (Proxy Server): 클라이언트와 서버 사이의 중계인 역할
    - Forward Proxy: 유저를 대리하여 서버에 접근하며 캐싱 및 필터링을 수행.
    - Reverse Proxy: 서버를 대리하여 로드 밸런싱(부하 분산) 및 보안(SSL 암호화)을 담당.
- 데이터베이스 분류
    - RDBMS (PostgreSQL 등): 엄격한 규칙과 정확성이 중요한 결제/회원 정보에 적합
    - NoSQL (DynamoDB 등): 자유로운 구조로 대량의 데이터를 빠르게 처리할 때 유리
- CAP 이론: 분산 시스템에서 일관성(C), 가용성(A), 분할 내성(P) 중 세 가지를 모두 만족할 수는 없다는 원칙
    - 네트워크 장애 시 데이터의 정확성(CP)을 택할지, 중단 없는 서비스(AP)를 택할지에 대한 설계적 선택이 필요

## 웹 서비스 아키텍처
- Web vs WAS
	- Web Server: 정적 컨텐츠(HTML, CSS, JS, 이미지)를 제공하여 누구에게나 동일한 결과를 보여줌
	- WAS(Web Application Server): DB 조회 및 비즈니스 로직을 처리하여 요청자에 따라 다른 결과를 생성하는 백엔드 영역
- 현대의 웹 동작 방식: React/Vue로 만든 뼈대를 먼저 받고, 백엔드와 JSON(API)를 주고받으며 동적으로 관리
- 학습 의의 : 서비스를 직접 배포,운영 ,모니터링 하기 위해 서비스가 어떻게 구성되는지 이해하는 기초 도구로서 학습

## HTML5: 의미와 구조
- HTML5 개발 원칙: 플러그인 최소화, 장치 독립성, 오류 처리 향상
- Semantic HTML(의미론적 웹): `<div>`만 쓰는게 아니라 태그 자체에 의미를 담아 이해하기 쉽게 만들기
	- SEO(검색 엔진 최적화)
	- 웹 접근성
- DOM(Document Object Model): HTML 문서를 브라우저가 이해할 수 있는 트리 구조의 객체로 변환한 것
- 웹 표준: W3C 등이 주관하며 모든 브라우저에서 동일하게 동작하도록 약속된 규칙

## CSS 기초와 Cascade
- CSS의 역할: HTML 요소가 어디에, 어떻게 보일지 정하는 규칙
- Cascade (우선순위 시스템):
	- 구체성(명시도) : 인라인 스타일 > ID > Class > Tag 순으로 우선권 존재
	- 선언 순서: 우선순위가 같을시 나중에 선언된 코드가 이전 코드를 덮어씀
- 적용 방식 : 외부 CSS 링크 방식이 유지보수와 일관성 면에서 권장

## Box Model 
레이아웃의 최소 단위
- 모든 요소는 박스다: 브라우저는 모든 요소를 가로/세로폭을 가진 사각형 박스로 인식
- 4겹 구조:
	1. Content: 실제 텍스트나 이미지가 표시되는 공간
	2. Padding: 내용과 테두리 사이의 안쪽 여백, 배경색이 여기가지 포함
	3. Border: 박스를 감싸는 경계선
	4. Margin: 박스와 박스 사이의 바깥쪽 여백
- box-sizing 속성:
	- content-box: 기본값, width가 content만 의미하여 계산이 복잡
	- border-box: width 안에 padding과 border를 포함시켜 레이아웃 예측이 쉬워짐

## 배치의 흐름을 바꾸는 도구
- 기본 문서 흐름: CSS를 적용하지 않으면 요소는 위에서 아래로, 왼쪽에서 오른쪽으로 흐르며, 이는 복잡한 UI 구성에 한계가 존재
- Flexbox(1차원 정렬): 부모가 자식 요소들의 정렬과 간격을 통제하는 시스템
	- 핵심 축: `flex-direction`에 따라 주축(Main Axis)와 교차축(Cross Axis)이 결정
	- 정렬: 주축은 `justify-content`, 교차축은 `align-items`로 정렬
	- 간격: `margin` 대신 `gap`을 사용하여 아이템 사이의 정식 간격을 처리
- Grid(2차원 설계): 화면을 격자로 나누어 행과 열을 동시에 설계
	- 비율 단위(`fr`): 고정된 px 대신 남은 공간의 비율로 칸을 나누어 반응형에 최적화
	- 용도: 카드 리스트, 갤러리 등 구조적인 대시보드 레이아웃에 사용
- Position: 일반적인 흐름을 벗어나 요소를 특정 위치에 고정하거나 겹칠때 사용
	- `relative`: 자기 자신을 기준으로 삼아 자식의 기준점이 됨
	- `absolute`:`relative`가 적용된 가장 가까운 부모를 기준으로 자유롭게 배치

## 반응형 웹과 인터랙티브 UI
- 반응형 웹 : 화면 크기에 따라 레이아웃이 조건부로 변하는 것
	- Media Query: `if`처럼 작용하며 특정 해상도 이하일 때 다른 CSS 적용
	- Viewport: 브라우저가 화면 폭을 계산하는 기준이며, 메타 태그 설정이 필수
- 의사 클래스 (Pseudo-class): 사용자의 행동에 따른 상태를 감지하여 스타일을 바꾼다.
	- `:hover`, `:focus`, `:active` 등 존재

## 개발 생산성 도구
Framework & Preprocessor
- CSS Framework: 미리 정의된 UI 규칙 세트, 디자인 통일성과 개발 속도를 높여줌
	- Bootstrap: 컴포넌트 기반으로 정해진 디자인을 빠르게 적용할 때 사용
	- Tailwind CSS: HTML 클래스 조합 방식으로 자유도가 높고 빠른 프로토타이핑 가능
- Boilerplate: 처음부터 만들 필요 없는 시작점인 기본 뼈대 코드 의미
- CSS Preprocessor(SASS/SCSS): 변수 사용, 코드 중첩, 파일 분리를 통해 CSS의 유지보수 제고
